Add auxiliary code to a playground

To add code that you don’t want to recompile each time you run your playground or you don’t want the user to see in the playground, add the source files to the playground or a page Sources folder. Xcode compiles files in Sources folders only when you add the files or save edits to the files.

The public symbols in files added to the playground Sources folder are available to all pages in the playground. The public symbols in files added to the page Sources folder are available only to the page containing the Sources folder.

Important: The compiled code imports into a playground as a module. To access a symbol (class, method, function, variable, or protocol) in the playground, the auxiliary Swift source file must export the symbol using the public keyword.

Add a new source file

  1. Open the Project navigator (choose View > Navigators > Show Project Navigator).

  2. In the Project navigator, select the Sources folder for the playground or a page, click the Add button (+) in the filter bar, then choose File from the pop-up menu (or choose File > New > File).

    A new Swift file appears in the Sources folder and opens in the source editor.

  3. Edit the code in the file, then choose File > Save (Command-S) to compile it.

Add an existing source file

  1. In the Project navigator, select the Sources folder for the playground or a page, click the Add button (+) in the filter bar, then choose Add Files to "Sources" from the pop-up menu (or choose File > New > Add Files to "Sources").

  2. In the dialog that appears, select the source file, then click Add.

    The file appears in the Sources folder.
