Configure app groups

Use app groups to allow multiple apps access to shared containers and allow additional interprocess communication between apps. First you enable app groups and then select the app group you want your app to belong to. If an existing app group is not sufficient, create a new app group.

Enable app groups

  1. In the project editor, select the target, then click Capabilities.

  2. In the App Groups section, click the switch to turn it from OFF to ON.

Select or deselect an app group

  1. In the Capabilities pane, click the App Groups disclosure triangle to reveal the settings.

  2. In the row of the container ID, select the checkbox to use the container and deselect the checkbox to not use the container.

    Xcode updates the list of app groups in the Xcode project entitlements file (a file with a .entitlements file extension).

Create an app group

  1. In the Capabilities pane, click the App Groups disclosure triangle to reveal the settings.

  2. At the bottom of the App Groups table, click the Add button (+).

  3. In the dialog that appears, enter a container ID in the text field, then click OK.

    An app group container ID begins with group. followed by a string in reverse DNS notation.

    Important: For macOS apps, the group container ID must be your Team ID followed by period and a unique name. Instead of a dialog appearing, a group container ID named $(TeamIdentifierPrefix). is added to the table. Double-click the container ID and append a subpath. The $(TeamIdentifierPrefix) variable will be replaced with your Team ID. For example, enter com.example.mygroup and the group container ID will be [Team ID].com.example.mygroup.
