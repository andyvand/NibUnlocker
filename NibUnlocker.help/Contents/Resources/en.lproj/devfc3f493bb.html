Configure Wallet (iOS, watchOS)

Wallet presents digital representations of information called passes—such as a coupon, ticket for a show, or boarding pass—that allow users to redeem a real-world product or service.

You can use Wallet in several ways:

Option 1: Request, distribute, and update passes

In your developer account, create a Pass Type ID certificate. You don’t need an app or an entitlement to do this but you'll need to register a pass type identifier first. For details, read Wallet Developer Guide.

Option 2: Let users add passes to Wallet from your app

You don’t need to configure Wallet in Xcode to do this. Use the PassKit framework described in Wallet Developer Guide.

Option 3: Access the user’s passes in your app

In Xcode, enable Wallet and in your developer account, register a pass type identifier. If you distribute multiple apps that access passes, you can limit your app to using a subset of your pass type identifiers.

Enable Wallet

  1. In the project editor, choose a target, then click Capabilities.

  2. In the Wallet section, click the switch to turn it from OFF to ON.

    Xcode will provision your app to use Wallet and add the PassKit framework to your project.

Limit an app to using a subset of pass type identifiers

  1. In the Capabilities pane, click the Wallet disclosure triangle to reveal the pass type identifiers options.

  2. Click the Refresh button under the Pass Types list to display recently created pass type identifiers.

  3. Select “Allow subset of pass types.”

    If there are no pass type identifiers in your developer account, the radio button reverts to “Allow all team pass types.” Register a pass type identifier before continuing.

  4. Select the pass type identifiers you want to use.
