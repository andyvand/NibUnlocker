Use workspaces for related projects

You can use a workspace as a container for several related projects. Xcode will detect implicit link dependencies between multiple targets in a workspace and build targets in the correct order. For example, Xcode builds a framework target before app targets which link to that framework.

Create a workspace

  1. Choose File > New > Workspace.

  2. Enter the workspace name, and specify its location in your file system.

  3. Click Save.

Add an existing project to a workspace

  1. In the Project navigator (), Control-click in the empty space below the list, then choose Add Files to “[Workspace]”.

  2. In the sheet that appears, select the project (a file with a .xcodeproj file extension), then click Add.

    The project appears in the Project navigator.

Alternatively, drag a project from Finder or a workspace window to the Project navigator. Make sure that you add the project at the root level, not as a subproject of another project.
