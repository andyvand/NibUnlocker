Switch the syntax coloring style of your code

By default, syntax coloring in the source editor is based on the file type of the currently displayed file. For example, Swift files use certain syntax colors, while Objective-C and property list files files use others. You can optionally switch to the syntax coloring of another programming language.

Set syntax coloring to a specific language: Choose Editor > Syntax Coloring > [Language].

Set syntax coloring to the default: Choose Editor > Syntax Coloring > Default for File Type.

Tip: You can set the global font and color scheme for Xcode in Preferences > Fonts & Colors.
