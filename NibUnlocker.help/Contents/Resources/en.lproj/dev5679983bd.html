Change a particle’s color

You can use the SpriteKit particle emitter editor to change the color of a particle throughout its lifetime and to determine how a particle blends in with other images.

Blend a particle’s texture with a color

The Color Blend fields control how the particle blends its color with its texture’s inherent color.

Factor: The average starting value of the blend factor. This value corresponds to the particle emitter’s BlendFactor property. The valid interval of values is from 0.0 up to and including 1.0. A value above or below that interval is clamped to the minimum (0.0) if below or the maximum (1.0) if above. The default value is 0.0, which means that the texture is used as is, ignoring the particle’s color. Otherwise, the texture is blended with the color.

Range: The range value creating a random variance of the starting color blend for each particle. This value corresponds to the particle emitter’s particleColorBlendFactorRange property.

Speed: The rate at which to modify the color blend for each particle, measured in change per second. This value corresponds to the particle emitter’s particleColorBlendFactorSpeed property.

  • Click the minus (—) or plus (+) button, or double-click in the field and enter the color blend values.

Change a particle’s color over its lifetime

  • Use the Color Ramp field to change the color of a particle throughout its life cycle. You can have a particle go through as many color changes as you want during a particle’s life cycle. The particle changes colors based on the space between color sliders.

    Add a new color slider to the color ramp: Click anywhere in the Color Ramp field.

    Create an immediate change of color: Place two color sliders on top of each other so that there is no space between them on the color ramp.

Blend particles with the background

Determine how each particle blends with other images in your app by selecting an option in the Blend Mode field. This value corresponds to SKBlendMode enumeration. The default value is SKBlendModeAlpha.

  • In the Blend mode pop-up field, select one of the options.

Change the alpha value

The particle’s color is the result of multiplying the particle’s alpha value with the texture and color blending state. The particle color is then blended with the parent’s framebuffer before the particle is displayed.

Start: The average starting transparency value applied to each particle. This value corresponds to the particle emitter’s particleAlpha property. The default value is 1.0.

Range: The range of allowed random values for the particle’s starting alpha value. This value corresponds to the particle emitter’s particleAlphaRange property. The default value is 0.0.

Speed: The rate at which the alpha value changes, measured in change per second. This value corresponds to the particle emitter’s particleAlphaSpeed property.

  • Click the minus (—) or plus (+) button, or double-click in the field and enter the alpha values.
