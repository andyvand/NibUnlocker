SpriteKit particles

A particle is similar to an SKSpriteNode object; it renders a textured or untextured image that is sized, colorized, and blended into the scene. However, particles differ from sprites in two important ways:

  • A particle’s texture is always stretched uniformly.

  • Particles are not represented by objects in the SpriteKit framework. This means you cannot perform node-related tasks on particles, nor can you associate physics bodies with particles to make them interact with other content. Although there is no visible class representing particles added by the emitter node, you can think of a particle as having properties like any other object.

The properties associated with SpriteKit particles are entirely defined by the emitter node that created the particles. For each property value, the SKEmitterNode class considers up to four factors:

  • The average starting value.

  • A range for the values. Each time a new particle is emitted, a random value is calculated within that range.

  • The rate at which the value changes over time, also known as the property’s speed. Not all properties have a speed property.

  • An optional keyframe sequence.
