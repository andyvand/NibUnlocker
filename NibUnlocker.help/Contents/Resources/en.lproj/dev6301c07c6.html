Reduce the size of your app by stripping Swift symbols

Xcode reduces the size of your app by stripping the symbols from Swift frameworks bundled with your app.

Why do I want Swift symbols stripped from my app?

In most cases the symbols are not required for Swift frameworks including the Swift standard library. Stripping out the symbols can make apps significantly smaller.

How much space does symbol stripping save?

The amount of space you will save depends on a few factors. To find the size difference, build a distribution version of your app with and without the symbols stripped, install each of them on a device, and compare the size. You can build a distribution version for a registered device, for manual distribution, or for TestFlight. Looking at the size of an archive build is not a reliable way to measure the size of an iOS, watchOS or tvOS app because the Swift standard libraries are only stripped as part of App Thinning.

How is Swift symbol stripping controlled?

Symbol stripping for a build target is controlled using build settings. The default settings depend on the target. For example, the default is to strip all symbols for the main app executable, but not to strip all symbols for a framework. Three build settings control how Swift symbols are stripped:

  • Strip Linked Product (STRIP_INSTALLED_PRODUCT) strips symbols from the build.

  • Strip Style (STRIP_STYLE) controls the level of symbol stripping.

  • Strip Swift Symbols (STRIP_SWIFT_SYMBOLS) overrides the strip style to remove all Swift symbols from the main app executable and from any bundled frameworks. This setting can make a significant difference for framework targets.

Is there a way to see the symbols after stripping?

Sometimes you may want to see the symbols for a framework that was previously stripped, such as when debugging a link failure. Because the linker uses a separate data structure to look up symbols in a framework, you can use the dyldinfo subcommand of the xcrun command-line tool to export the symbols by running xcrun dyldinfo -export in the terminal.

Why would I turn symbol stripping off?

Symbols stripped during the build process can be saved to a dSYM file which Xcode can use to read the symbols at a later date. There are a few cases when the dSYM file is not sufficient. In addition, there is no dSYM file for the Swift standard library. The cases include:

  • Enterprise apps in the QA cycle when the symbols from the Swift standard library are needed for crash logs.

  • Distribution of the app without using the AppStore or TestFlight.

  • Using some third-party crash logging tools.

  • Using the dladdr system function for in-process crash symbolication.

Most of the time you can reenable symbol stripping for your final app.
