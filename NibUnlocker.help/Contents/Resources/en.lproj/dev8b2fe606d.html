Assign and configure custom shaders

Shader objects are used to customize the drawing behavior of many different kinds of nodes in SpriteKit. A custom shader uses a fragment shader file to adjust the rendering or filtering of particle emitters. To provide uniform values to the shader, create uniform objects in the SpriteKit particle emitter editor.

Add a custom shader

A fragment shader’s source file (with a .fsh extension) contains code that creates particle effects. Use a custom shader to apply that code to your particle emitter.

Before adding a custom shader to your Xcode project, create your fragment shader file in the Metal Shading Language Guide or the OpenGL ES shading language.

  1. Add the fragment shader source file to your project.

  2. Select your particle emitter file in the navigator area.

    Xcode opens the SpriteKit particle emitter editor.

  3. Open the Attributes inspector.

  4. In the Custom Shader pop-up menu, choose your custom shader file.

    Xcode creates a new SKShader object by loading the source for a fragment shader from the .fsh file. The particle emitter’s shader property points to the new SKShader object.

Create custom shader uniforms

An SKUniform object is used to hold varied types of data to be exchanged with a custom shader. The Custom Shader Uniforms area lets you create uniforms to be used in your shader without declaring the uniforms' value or type in the source.

  1. Click the plus (+) button at the bottom of the Custom Shader Uniforms area.

    Xcode creates a new uniform object with default values and displays the values in the Custom Shader Uniforms area.

  2. To change the uniform’s name, click on the current name and enter a new value.

    Important: The name you enter must match the name used in the fragment shader source.

  3. In the Type pop-up menu, chose a type.

    The available types are described in the SKUniformType enumeration.

  4. To change the uniform’s value, click on the existing value and enter a new value.
