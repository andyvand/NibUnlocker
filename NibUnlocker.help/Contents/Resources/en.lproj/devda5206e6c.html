Import localizations

After you export localizations and are done translating the files, you import the files back into your Xcode project.

  1. In the Project navigator, choose the project, then choose Editor > Import Localizations.

  2. In the import dialog that appears, select the Xcode Localization Catalog (xcloc) (a folder with a .xcloc file extension), then click Open.

    Xcode ingests the files and warns you if some files havenâ€™t been translated.

    Note: If you exported localizations using a previous version of Xcode, you can also select an XML Localization Interchange File Format (XLIFF) file.

  3. In the sheet that appears, review the changes, warnings, and errors.

    In the left column, click the warning or error icon in the toolbar, then select a message that appears below. In the comparison editor, the current version of the file appears on the left and the imported version on the right.

  4. To review the changes, click the file icon in the toolbar of the sheet, then select a file in the navigator below.

  5. Click Import.

Xcode generates the strings files and stringsdict files from the translations provided in the XLIFF files.

Alternatively, you can import both xcloc and XLIFF folders using the xcodebuild command with the importLocalizations option:

xcodebuild -importLocalizations -project <projectname> -localizationPath <dirpath>
