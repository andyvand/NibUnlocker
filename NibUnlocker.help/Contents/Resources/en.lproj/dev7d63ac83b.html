event actions

Xcode can initiate the following actions when certain types of events occur, such as a build starts, generates issues, succeeds, or fails. Actions are configured in Xcode > Preferences > Behaviors.

Action

Description

Play sound

To pick a sound file to play, choose a system sound from the pop-up menu or select Choose Sound. Supported sound file formats include .aiff, .m4a, .mp3, and .wav.

Speak announcement using

Choose a voice to speak a notification, such as “Build Succeeded” or “Build Failed.”

Notify using bezel or system notification

Enable a brief visual notification, such as “Build Succeeded” or “Build Failed,” centered on the main screen above other applications.

Bounce Xcode icon in Dock if application inactive

Cause the Xcode icon to bounce in the dock if Xcode is not the active app.

Show tab named

Activate the specified tab. You can enter the name of any tab in an open main window. If the named tab doesn’t exist, Xcode creates it in the active main window.

Show/hide navigator

Show or hide the specified navigator. Options include the current navigator, Project navigator, Symbol navigator, Find navigator, Issue navigator, Test navigator, Debug navigator, Breakpoint navigator, and Report navigator.

Show/hide debugger

Show the debug area, hide it, or hide it only if your code has not sent any output to standard out or standard error. When showing the debug area, you can choose to show your current debug area views, the variables view, the console view, or both the variables and console views.

Show/hide utilities

Show or hide the utilities pane.

Show/hide toolbar

Show or hide the toolbar.

Show/hide editor

Show or hide the specified editor area. Options include the current editor, standard editor, assistant editor, or version editor.

Navigate to

Navigate to one of the following locations when a build issue is encountered:

  • The first new issue in the Issue navigator.

  • The build log in the Report navigator.

Note that this action changes keyboard focus.

Run

Run an executable script file and wait for it to finish running before proceeding with the action.
