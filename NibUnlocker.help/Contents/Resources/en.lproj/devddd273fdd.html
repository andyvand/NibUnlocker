Set supported document types (iOS, macOS)

You can configure your app to support different document types. You can edit the document types setting in the Info pane for the target, or by editing the information property list directly.

Go to CFBundleDocumentTypes in Information Property List Key Reference for the possible values for the CFBundleTypeName (Name field), LSItemContentTypes (Types field), CFBundleTypeRole, and other keys that appear in the Info.plist file.

  1. In the Project navigator (), choose the target, then click Info.

  2. Click the disclosure triangle next to Document Types to reveal the settings.

  3. To add a new document type, click the Add button (+) at the bottom of this section.

  4. In the Name field, enter the document type’s name (CFBundleTypeName) and in the Types field, enter its uniform type identifier (LSItemContentTypes).

  5. Click the “Additional document type properties” disclosure triangle to reveal the Key table.

  6. To add a key-value pair, click the text below the Key table, then enter the key and its value.

    For macOS apps, enter CFBundleTypeRole as the key and in the Value column, enter a document role.

  7. To add another key-value pair, click the Add button (+) in a row, then enter a key-value pair in the new row. To delete a key-value pair, click the Delete button (–).

If you view the Info.plist file in the source editor, the CFBundleDocumentTypes dictionary should appear similar to this:

<key>CFBundleDocumentTypes</key><array> <dict> <key>CFBundleTypeIconFiles</key> <array/> <key>CFBundleTypeName</key> <string>Text</string> <key>LSHandlerRank</key> <string>Alternate</string> <key>LSItemContentTypes</key> <array> <string>public.plain-text</string> </array> </dict></array>
