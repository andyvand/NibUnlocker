Add simulated status, top, and bottom bars

Add or remove a simulated status, top, or bottom bar to override the default display of these items. The simulated user interface elements display in other view elements if that is appropriate. For example, add a top bar to a view controller that is part of a navigation controller hierarchy on a different storyboard.

  1. On the Interface Builder canvas or in the outline view, select the desired view controller.

  2. Choose View > Utilities > Show Attributes Inspector.

    The Attributes inspector for the selected controller appears in the inspector area. The inspector includes the Simulated Metrics area with choices for the Status, Top, and Bottom Bars as shown below:

  3. Click the pop-up menu for the bar you want to configure.

    A pop-up menu opens showing the possible configurations for that bar. For example, the pop-up menu for the top bar is shown below:

  4. Choose the item that best matches your desired configuration.

    Every pop-up menu has items for None and Inferred. The remaining items vary depending on the type of bar.

    None: Hides the bar so it does not appear in the controller.

    Inferred: Shows or hides the bar depending on the type of controller. This is the default setting.

Changing type of simulated status, top, or bottom bar for a view controller changes the type for child view controllers that have a simulated type of “Inferred”.
