Reference a project from another project

A project can include a cross-project reference to another project for the purposes of establishing a dependency and referring to the other project’s targets, files, and products.

  1. Open the Project navigator.

  2. Click the Add button (+) at the bottom of the Project navigator.

  3. Choose Add Files to [ProjectName] from the menu that appears.

  4. Select the desired project folder.

  5. Click Add.

After adding a cross-project reference to a project, you can refer to the other project’s targets, files, and products using the Target Dependencies, Copy Files, and Link Binary With Libraries build phases. If the other project can’t be located at build time, the build system still attempts to build the product, but produces missing dependency warnings.

Note: It’s generally good practice to use a workspace to manage multiple projects. However, projects defined within workspaces can’t define explicit dependencies on one another.
