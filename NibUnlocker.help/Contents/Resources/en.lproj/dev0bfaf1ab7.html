Use texture atlases to improve performance

A texture atlas improves performance of your apps by combining multiple image assets into a larger image file. Then your app can draw multiple images with a single draw call.

The compiler looks for folders with a .atlas file extension in your project and combines the containing images into one or more texture atlases. The images are rotated and trimmed to fit the maximum number of images into each texture atlas. The maximum size of a texture atlas image is 2048 x 2048 or 4098 x 4098 pixels, depending on your project’s texture atlas options. If a texture atlas image reaches the maximum size, the compiler creates a new texture atlas.

In the app bundle’s Resource folder, a texture atlas is represented by a folder with a .atlasc file extension. The texture atlas images and their orientation are tracked by an associated property list. The SpriteKit framework knows how to access your images through the property list and therefore, no code changes are required.

If your project doesn’t use the SpriteKit framework, you can still use texture atlases by including .atlas folders in your project and manually reading the generated property list file at runtime.

Add a texture atlas to your project

  1. Place the image files you want to add to a texture atlas in a single folder.

  2. Give the folder a .atlas file extension.

  3. Choose File > Add Files to [AppName].

Change the texture atlas options

  1. In the Project navigator, select your project.

  2. In the editor area, click Build Settings.

  3. Type texture in the search field, and press Return.

    The SpriteKit Texture Atlas Options appear.

    Important: To view the texture atlas options, you must have a folder with a .atlas file extension in your project.

  4. From the Texture Atlas Maximum Size pop-up menu, choose a maximum size.
