Add language plural variants

To create plural variants of your localizable strings that you pass to the NSLocalizedString macro in your code, you add a stringsdict file (a file with a .stringsdict file extension) to your project.

For example, you may need a stringsdict file for the formatted string “%d files remaining”. In English, if the value is 1 for the variable d, then the variant is “1 file remaining” or “one file remaining”; otherwise, the variant is “%d files remaining”. Other languages may have more plural variants for this formatted string.

Add a stringsdict file from a template

  1. Choose File > New > File.

  2. In the sheet that appears, select the target OS.

  3. Under Resources, select Stringsdict File, then click Next.

  4. In the sheet that appears, enter a filename, choose a location for the file, then click Create.

Edit the stringsdict file template

The template contains a dictionary with the plural variants for a single formatted string. For each formatted string you want to define plural variants, duplicate the Localized String Key (StringKey)-value pair provided in the template. Then make the following edits for each formatted string:

  1. Replace Localized String Key (StringKey) with the formatted string you pass to the NSLocalizedString macro.

  2. Replace VARIABLE (in two locations) with the name of the variable you use in the localizable string, the value for Localized Format Key (NSStringLocalizedFormatKey).

    Note: If you have more than one variable in the Localized Format Key string, duplicate the VARIABLE key-value pair for each variable.

  3. For each plural variant required for your development language, enter a formatted string for the corresponding category, then remove the categories you don’t use.

    For example, enter strings for the one and other categories, then remove the zero and two categories for English.

    Important: You provide a value for the other category which is required.

If you edit the stringsdict file using the property list editor (the default editor for a stringsdict file), the Xcode titles for the keys appear (for example, Localized Format Key). If you view the file in the source editor (Control-click the file and choose Open As > Source Code), the raw keys appear (for example, NSStringLocalizedFormatKey).
