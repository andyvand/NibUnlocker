Distribution options

You select various distribution options when you export your app depending on the distribution method you choose, the platform, the capabilities you enable, and other features you use.

Ad Hoc, Enterprise, and Development options

Option

Description

iCloud Environment

If the app uses iCloud Documents or CloudKit, choose either the Development or Production environment. If you choose Production, the app accesses the same environment as the version of your app sold on the App Store.

App Thinning

Choose whether to create one or more variants of the app for specific devices.

  • None: Create a single app that runs on all supported devices, called the universal variant.

  • All compatible device variants: Create variants for all compatible devices, and a universal variant.

  • A specific device: Create variants for the selected device.

To learn more, go to What is app thinning? (iOS, tvOS, watchOS).

On-Demand Resources

Choose how to handle on-demand resources:

  • Embed in App: Places all the on-demand resources in the app bundle.

  • Host on Server: You expect to host the on-demand resources yourself. If you choose this option, enter the URL for your server.

Rebuild from Bitcode

Creates the app from bitcode similar to how the App Store will create builds from bitcode.

Include manifest for over-the-air installation

If you want to host the resources yourself when distributing your app outside the App Store, select this option to include a manifest file.

App Store options

Option

Description

Include bitcode for iOS content

Includes bitcode which allows the App Store to compile your app optimized for the target devices, and may recompile it later to take advantage of specific hardware, software, or compiler changes.

This option appears for iOS targets only. tvOS and watchOS apps automatically include bitcode.
