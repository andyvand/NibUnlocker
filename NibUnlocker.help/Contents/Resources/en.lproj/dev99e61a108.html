build dependency

A build dependency is the reliance of one target on the built products of another target. In some cases, Xcode can discover these dependencies and build the targets in the appropriate order. For example, when an app target links to a library target, Xcode can infer that the library target must be built first. Such a relationship is an implicit dependency. You can also define an explicit dependency on another target in the target dependencies build phase of a target. In this case, you specify the order in which the other target is built.

Note: The Find Implicit Dependencies checkbox must be selected in the Build pane of the scheme editor for Xcode to infer target dependencies.
