code indentation options

The following code indentation options are configurable in Xcode > Preferences > Text Editing > Editing:

Option

Description

Prefer indent using

The preferred characters—spaces or tabs—to use when indenting code. Note that choosing tabs may still result in spaces if the tab width and indent width don’t match.

Tab width

The visual width of tab characters, in spaces.

Indent width

The number of spaces used to indent blocks of code.

Tab key

The behavior of the tab key for indentation. Options include: “Indents in leading whitespace,” “Indents always,” and “Inserts tab character.”

Wrap lines to editor width

Cause lines to wrap when they reach the width of the source editor.

Indent wrapped lines by

The number of leading spaces to display before wrapped lines. Note that this is for visual purposes only and spaces are not actually inserted into the wrapped lines.

Syntax-aware indenting

Automatically indent code when typing or pasting, based on the code’s syntax.

Indent solo "{"

Apply indentation to opening braces ("{") that reside on a line by themselves.

Indent // comments one level deeper

Indent single-line comments one level deeper than the surrounding code.

Align consecutive // comments

Indent consecutive single-line comments to match the indentation level of the first single-line comment in the series.

Automatic indent for

Automatically indent when you type certain characters. Options include: Left Brace ("{"), Right Brace ("}"), Semicolon (";"), Colon (":"), Hash Mark ("#"), and Return.
