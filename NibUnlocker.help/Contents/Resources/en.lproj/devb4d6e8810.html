Configure actions for events

Use Behaviors preferences to configure Xcode to respond to certain events by performing one or more specific actions, such as playing a sound, displaying a notification, opening a tab, or running a script.

Note: Not all actions are valid or meaningful for every event. When one or more actions are enabled for an event, a check mark appears before the event name.

Set actions for an event

  1. In the left column, select an event.

  2. In the right column of the pane, select one or more action checkboxes and configure any corresponding options.

Set up a custom tab to open automatically when an event occurs

  1. In the left column, select an event.

  2. In the right column of the pane, select the “Show tab named” checkbox.

  3. Enter a tab name.

  4. Choose whether the tab should be opened in the active window or a new window.

  5. Enable and configure any additional actions, as needed.

Create a custom behavior

  1. At the bottom of the left column, click the Add button (+).

  2. Type a name for the custom behavior.

  3. Optionally, double-click the icon to the right of the custom behavior name and type a keyboard shortcut.

  4. In the right column of the pane, select one or more action checkboxes and configure any corresponding options.

Initiate a custom behavior

  • Choose Xcode > Behaviors > [Custom Behavior Name] or press the keyboard shortcut for the behavior, if one exists.

Remove a custom behavior

  1. In the left column, select a custom event.

  2. At the bottom of the left column of the pane, click the Delete button (–).
