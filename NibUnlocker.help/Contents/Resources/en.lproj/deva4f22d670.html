If you use the deprecated Always Search User Paths build setting

The Legacy Build System supports the Always Search User Paths build setting, which causes Xcode to always search user header search paths before system header search paths, even when using #include <header.h>-style includes, and to find user headers located on disk without requiring the Xcode project to explicitly reference them. This feature has been deprecated since Xcode 8.3 and is not supported by the New Build System. If your project depends on this feature, you may need to modify your build settings or source files to stop using it.

If the build warns that "Traditional headermap style is no longer supported”

When you build a target which has the "Always Search User Paths" build setting enabled, you will see a warning similar to:

warning: Traditional headermap style is no longer supported; please migrate to using separate headermaps and set 'ALWAYS_SEARCH_USER_PATHS' to NO.

  • To eliminate this warning, set the ALWAYS_SEARCH_USER_PATHS build setting to NO as instructed.

    This will disable the warning. If your project was relying on this feature and encounters further build errors, you may need to perform further troubleshooting steps.

If a build error is similar to "error: 'filename.h' file not found with <angled> include; use "quotes" instead”

This error can indicate that filename.h is being included in a file with a directive like #import <filename.h>. Angle includes such as this are intended for framework-style includes, such as <Foundation/Foundation.h>, or for including system headers, such as <string.h>.

  • In this case, filename.h is being included directly as part of the project, so change the directive to use a quote include (#import "filename.h") instead.

If a build error is similar to `fatal error: 'filename.h' file not found`

This may indicate that the header which cannot be found is not referenced by your project.

To fix this problem, follow these steps.

  1. If the header exists on disk, add the header to the project as a file reference. To create a file reference, unselect the “Copy items if needed” checkbox when you add existing files and folders.

  2. If, as a result of adding the file reference, the project contains multiple headers with the same name, add the header to the target.
